# Программа работает с cel-файлами, обработанными утилитой от Rebecca Heineman, cel2shp. После такой обработки CCB флаги будут располагаться в первых четырех байтах обрезанного cel-файла.
# Каждая группа спрайта начинается командой SPRITEBEGIN и заканчивается командой SPRITEEND.
# Префиксы пишу в десятичном формате, т.е. префикс 0x40 = 64, 0x80 = 128.
# ADDOFFSET дает инструкцию паковать спрайты как для вражин, т.е. с префиксами 0x4000 и суммируя указатели.
#NEXTLINE для контроля строки двумерного массива,  0x4000.
#NEXTGROUP для контроля столбца многомерного массива,  0x4000.
# В данной версии программы максимальный размер массива 8х8.
# Команда MIRRORED не сохраняет данный кадр в файл спрайта т.к. он уже есть в зеркальном отображении, и прописывает на него смещение с префиксом 0х8000, что означает что кадр зеркален.
# Для вражин обязательно подготавливать кадры по 8 шт, 3 последних из которых являются зеркальными. Группы по 8 кадров указываются параметром команды ADDOFFSET.
# Обычные кадры дописываются после групп 8-ок. Обычных кадров может не быть. Обычные кадры могут являться кадрами смерти, шаров и прочего. 
# В основе работы программы лежит чтение CCB флагов. Все Cel-файлы, присутствующие в скрипт-файле должны иметь одинаковые флаги.
# Если при расчете появляется такое сообщение:
# Ordinary offset is greater than 0xFFFF. Frame# so Prefix value is not allowed! Значит фрейм зеркалиться не будет. Нужно сделать отдельный файл и убрать префикс из скрипта.
# Команда MIRROR_OFF указывает, что ВСЕ группы кадров по 8 шт. ниже являются НЕ зеркальными.
